=================================================== CLEAN ARCHITECTURE ===================================================

É um padrão de arquitetura de software proposto por Robert C. Martin que visa criar sistemas de software bem estruturados,
altamente testáveis e de fácil manutenção sendo focada na separação de responsabilidades e na criação de camadas indepen -
dentes, permitindo que o código seja mais flexível e adaptável a mudanças.

Oficialmente é dividida em circulos:

- APPLICATION atua como uma ponte entre a camada de domínio e as interfaces externas da aplicação

OBS: Infrastructure e Persistence pode ser tudo Infrastructure...
OBS: Não é super rigido a separação, varia de cenário para cenário, mas essa é a base.

círculo mais externo{
	Presentation (Interface Adapters):
		- Web Pages
		- Web Components
		- Web APIs
		- Controllers
		- Views
		- Filters
		- Attributes
		- View Models
		- Style Sheets
		- JavaScript

	Infrastructure (Frameworks & Drivers):
		- Auth e Identity
		- File e object Storage
		- Message Queue Storage
		- Services
		- Email/Notification Services
		- Logging Services
		- Paymant Services
		- Social Logins

	Persistence (Frameworks & Drivers):
		- Data Context
		- Repositories
		- Data Migrations
		- Data Seeding
		- In Memory caching
		- Cache
}

Existe um conceito claro acerca das dependências:
- É de fora para dentro.
- Nada do circulo interno sabe do circulo externo, mas o externo sabe do interno.
*Analisar as dependências do projeto para melhor compreensão*
	- Domain não possui nenhuma dependência.
	- Application depende do Domain.
	- Persistence depende do Application.
	- API depende do Application e Persistence(Somente em tempo de execução).
