=================================================== POSSIBILIDADES ===================================================

Como usei os conceitos de CQRS e mediator junto da biblioteca MediatR e de forma simples alguns possível objetos fica-
ram de fora.

==================================================== BATE PRONTO =====================================================
DTOS -> APPLICATION.
OPTIONS -> INFRASTRUCTURE.
EXTENSIONS -> INFRASTRUCTURE.
PROVIDERS -> INFRASTRUCTURE.
IPROVIDERS -> APPLICATION (Interfaces de providers).
AUTHHANDLER + REQUIREMENT -> PRESENTATION.
FILES -> INFRASTRUCTURE.
MIDDLEWARE (EXCEPTION/AUTH/ANOTHER) -> PRESENTATION.

EXCEPTIONS devem ser avaliadas as finalidades de cada uma
	- Exceptions de negócio -> DOMAIN
	- Exceptions de fluxo inválido, não encontrado, conflito -> APPLICATION.
	- Exception de erro com DB, erro API externa -> INFRASTRUCTURE.

CONSTANTS devem ser avaliadas as finalidades de cada uma 
	- Mensagem de erro na validação -> APPLICATION.
	- Mensagem de regra de negócio quebrada -> DOMAIN.
	- Conceitos de negócio/domínio -> DOMAIN.
	- Texto de Api -> PRESENTATION.

ENUMS devem ser avaliadas as finalidades de cada uma
	- Status/Indicador (banco) -> DOMAIN.
	- Fluxo -> APPLICATION.

FACTORIES devem ser avaliadas as finalidades de cada uma
	- Normalmente Interfaces e implementação -> APPLICATION
	- Factory de externalService (Email) -> Interface (APPLICATION) / implementação (INFRASTRUCTURE)


==================================================== ORGANIZADO ====================================================

DOMAIN:
	- Exceptions de negócio
	- Constant de Mensagem de regra de negócio quebrada.
	- Constant de  Conceitos de negócio/domínio.
	- Enum de Status/Indicador (banco).

APPLICATION:
	- DTOs.
	- Exceptions de fluxo inválido, não encontrado, conflito.
	- Constant de Mensagem de erro na validação.
	- Enum de fluxo.
	- Factories Sempre Interfaces e normalmente implementação.

INFRASTRUCTURE:
	- Options.
	- Extensions.
	- Files.
	- Exception de erro com DB, erro API externa.
	- Factories de external services (IMPLEMENTAÇÃO SOMENTE).

PRESENTATION:
	- Middlewares (Exception/Auth).
	- Auth Handler + Requirement.
	- Constant de Texto de API.
	- Provider que dependde de HttpContext, Claim, Request...
	- Extensions de HttpContext, Claim, Request...
